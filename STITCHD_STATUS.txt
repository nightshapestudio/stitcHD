====================================================
PRODUCT IDENTITY
====================================================

STITCHD has TWO equally core modes. Neither is secondary.

MODE 1 — MULTI-TRACK RECONSTRUCTION
- Import multiple versions of a song or stems
- Align on a shared timeline
- Replace broken/weak sections with clean ones from alternate versions
- Build a Frankenstein master from the best parts

MODE 2 — SINGLE-TRACK TEMPO REPAIR
- Import one track
- Detect average BPM
- Generate a beat/bar grid
- Segment the song into sections
- Tighten timing drift across the arrangement
- Repair inconsistent pacing section by section
- Export a tightened final WAV
- No additional tracks required

Single-track repair philosophy:
- Preserve pitch, feel, groove, transients, vocal quality
- Behave like intelligent tempo conforming and timing micro-surgery
- NOT crude global speed change or pitch-altering playbackRate stretch

====================================================
ARCHITECTURE NOTES
====================================================

- STITCHD is entirely client-side: Web Audio API, Canvas, Zustand, no DB or API server.
- Clip arrangement lane uses pointer capture API for smooth drag.
- WAV export uses OfflineAudioContext at 44.1 kHz or 48 kHz, 16-bit PCM stereo.
- Beat grid snapping uses Shift key modifier on drag operations.
- vite.config.ts uses dedupe for react, react-dom, zustand.
